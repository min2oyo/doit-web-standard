시맨틱 태그(semantic tag)

<header>
- 주로 맨 위쪽이나 왼쪽에 위치
- 검색 창이나 사이트 메뉴를 삽입

    <nav>
    - 내비게이션 영역
    - 같은 웹 문서 안에서 다른 위치로 연결
    - 다른 웹 문서로 연결하는 링크를 만듦
    - 웹 문서의 위치에 영향을 받지 않음
        - <header>, <footer>, 사이드 바 안에 포함되거나 독립할 수 있음
    - 여러개 사용할 경우 id 속성을 지정

<main>
- 핵심 콘텐츠를 담음
- 웹 문서마다 다르게 보여주는 내용
- 한 번만 사용할 수 있음

    <article>
    - 독립적인 콘텐츠로 담음(블로그의 포스트, 뉴스 기사)
    - 사전 의미인 신문이나 잡지의 기사처럼 웹에서 실제로 보여주고 싶은 내용을 넣음
    - 문서 안에 여러 개 사용할 수 있음
    - 이 안에 <section> 태그를 넣을 수 있음

    <section>
    - 큰텐츠 영역을 나타냄
    - 몇 개의 콘텐츠를 묶는 용도로 사용
    - 단순히 스타일을 적용하려고 콘텐츠를 묶으려면 <div>태그 대신 사용

<aside>
- 사이드 바 영역을 나타냄
- 본문 내용 외에 왼쪽이나 오른쪽, 혹은 아래쪽에 사이드 바를 만듦
- 필수 요소는 아님

<footer>
- 웹 문서 맨 아래쪽
- 사이트 제작정보, 저작권 정보, 연락처 등을 넣음
- <header>, <section>, <article> 등 다른 시맨틱 태그를 모두 사용할 수 있음
- SNS 링크나 연락처 등의 정보를 넣기도 함

<div>
- division
- 여러 요소를 묶음
- <div id="header">나 <div class="detail">처럼 id나 class 속성을 사용해서 문서 구조를 만들거나 스타일을 적용할 때 사용함
